SmartBot Functionality & Installation Guide

Overview

SmartBot is a Swing-based desktop application that simulates a chatbot interaction interface. The bot can handle text and file input, store and display chat history, and generate PDF reports from the chat content. It also supports attaching and processing files like images.

Functionality Breakdown

1. UI Components

The main UI components include:

  • JTextField: Used for entering text input.
  • JEditorPane: Displays the chat history in HTML format.
  • JButton: Buttons for sending messages, clearing messages, and attaching files.
  • JPanel: Organizes the layout of different components.
  • JFileChooser: Facilitates file selection for attachments.
  • JScrollPane: Enables scrolling within the chat history panel.

2. Chat Functionality

The chat interface allows users to input text and interact with the bot. Key features include:

  • Text input with Enter key press to send messages.
  • Option to attach files, which can be processed and included in the chat.
  • Chat history is displayed in an HTML format with a timestamp and stored locally in an HTML file.

3. File Attachment and Processing

Users can attach multiple files (PDFs or images) to the chat. The bot processes these files, displaying the images and storing the PDF content. Additionally, users can right-click on the chat history to generate a PDF report of the conversation.

4. Model Integration

The application supports interaction with a selected language model (e.g., "llava") using the OllamaStreamingChatModel. The response from the model is streamed and displayed in real-time within the chat interface.

Installation Guide

1. Prerequisites

  • Java Development Kit (JDK) 17 or above
  • Maven or Gradle build tool
  • Internet connection (for downloading dependencies)
  • Ollama Installation

2. Setup Instructions

Follow the steps below to set up and run the SmartBot application:

  1. Clone the Repository: Clone the project repository from GitHub or download the source code.
  2. Open the Project: Open the project in your preferred IDE (e.g., IntelliJ IDEA, Eclipse).
  3. Install Dependencies: Use Maven or Gradle to download the required dependencies. Run the following command in the terminal: 
    mvn install
  4. Configure the API Endpoint: Ensure that the URL in the SmartBot class is correctly pointing to the API endpoint for the language model.
  5. Run the Application: Run the SmartBot class as a Java application. The UI should launch, allowing you to interact with the bot.
  6. File Storage: The chat history is saved in the "chat" directory within the project's root directory.

3. Troubleshooting

  • If the application fails to start, check that all dependencies are correctly installed.
  • If the chat history is not saving, ensure that the "chat" directory has the appropriate write permissions.
  • For the pdf book creation, check for the write permission for output directories
  • If file attachments are not processed, verify that the correct file types are being selected (JPG, PNG).

Conclusion

With these steps, you should have SmartBot up and running. The application provides a robust foundation for building and extending chatbot capabilities, especially for integrating AI language models.
